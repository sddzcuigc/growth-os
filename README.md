# 成长OS

成长OS是一套“通过多情境证据理解孩子，再用真实项目验证判断”的成长系统，而不是让家长先选技能、再拼任务的配置工具。

## 当前版本：V5.1 结构化访谈

主流程：

1. 全面了解
2. 形成阶段性发现
3. 映射未来技能路线
4. 自动策划项目
5. 执行、留证据并反向修正画像

- GitHub Pages：`https://sddzcuigc.github.io/growth-os/`
- Vercel：`https://growth-os-ten-pearl.vercel.app`

## 一、全面了解：不再单选

V5.1删除了“点一个快捷答案就进入下一题”的单选式流程。

每个主题都由五部分组成：

- 多选行为观察
- 出现频率
- 主要情境，可多选
- 信息来源
- 具体事例与补充说明

同一个孩子可以同时选择看似矛盾的行为，例如：

- 喜欢的事情能长时间投入
- 普通作业却需要多次催促
- 在学校能独立完成
- 在家容易依赖成人陪伴

系统不会把矛盾项互相抵消，而会标记为“情境差异”，继续分析任务难度、兴趣程度、成人介入方式和环境影响。

## 二、九个结构化主题

访谈不再由随意想到的问题组成，而是固定覆盖九个互补维度：

1. 自主兴趣与内在动机
2. 学习、理解与解决问题方式
3. 任务执行与自我管理
4. 表达、创造与成果呈现
5. 情绪、挫折与恢复
6. 合作、边界与同伴关系
7. 身体状态与生活节律
8. 家庭环境、资源与现实限制
9. 半年目标与优先级

每个主题可以重复补充，也可以由不同观察者分别提交。

## 三、多来源证据

每条记录都可以标明来自：

- 家长观察
- 孩子自述
- 老师反馈
- 老人或其他照料者
- 教练或兴趣老师

系统会统计：

- 主题覆盖数量
- 信息来源数量
- 观察情境数量
- 具体事例数量

只有在不同时间、不同情境或不同观察者之间重复出现的行为，置信度才会提高。

## 四、发现报告

报告把信息分为：

- 已明确观察到的事实
- 优势与天赋苗头
- 当前最值得支持的能力
- 矛盾与情境差异
- 仍然不知道、需要继续补充的信息

每条优势或困难都会显示：

- 证据来自谁
- 出现在哪里
- 发生频率
- 对应具体事例
- 当前置信度

“天赋”不会由一次勾选直接确认。只有当优势跨时间、跨情境、低催促地重复出现，并能形成真实成果时，才逐步提高判断置信度。

## 五、未来技能路线

系统自动把证据映射到成长OS技能树：

1. 利用优势：让孩子愿意进入项目
2. 核心补强：一次只抓1—3项关键能力
3. 重要支撑：在项目中自然发生
4. 基础维护：每个项目持续检查

技能方向包括：

- 责任闭环
- 任务拆解
- 深度阅读
- 中文书写
- 数据分析
- 文件管理
- 信息可信度
- 计算思维
- AI协作
- 用户意识
- 合作分工
- 挫折恢复
- 反馈与提炼
- 迁移应用

## 六、自动策划项目

系统结合以下信息推荐三个项目：

- 稳定兴趣和优势线索
- 当前主要支持需求
- 半年目标
- 家庭资源与限制
- 可用时间
- 真实使用者

每个推荐会解释：

- 利用了什么优势
- 重点支持什么能力
- 对应哪些未来技能
- 为什么适合当前阶段

家长仍可通过“高级调整”修改外壳、核心技能、真实使用者和时间。

## 七、执行与反向验证

项目不是课程表，而是一次验证假设的实验。

执行中记录：

- 哪些情境表现更好
- 成人介入多少
- 卡点发生在哪一环
- 是否能自己恢复
- 是否检查和收尾
- 真实使用者如何反馈
- 是否根据反馈修改
- 能否迁移到新场景

这些证据会反过来修正孩子画像和下一轮技能优先级。

## 当前能力边界

V5.1使用浏览器本地的可解释访谈与证据聚合内核。它能完成多选记录、多来源对比、情境差异识别、置信度分层、技能映射和项目推荐。

当前访谈问题仍由结构化规则控制，不是完全自由的大模型追问。后续可让多供应商API或自定义GPT根据已有回答生成更深入的追问；本地规则继续作为可解释回退。

## Plus Action自动桥接

Plus Action继续用于深度项目规划：

1. 成长OS创建规划请求
2. 自动打开用户自己的自定义GPT
3. 用户发送一次任务编号
4. GPT读取孩子发现、技能路线和项目建议
5. GPT生成完整项目并写回
6. 成长OS轮询、导入并进入执行

成长OS不能替代用户在ChatGPT界面点击发送，因此仍需要用户发送一次任务命令。

## Vercel桥接环境变量

必须配置：

- `GROWTH_OS_BRIDGE_KEY`
- Redis REST地址与Token，支持以下任一组：
  - `UPSTASH_REDIS_REST_URL` + `UPSTASH_REDIS_REST_TOKEN`
  - `KV_REST_API_URL` + `KV_REST_API_TOKEN`
  - `REDIS_REST_URL` + `REDIS_REST_TOKEN`
- `ALLOWED_ORIGINS`：建议填写 `https://sddzcuigc.github.io,https://growth-os-ten-pearl.vercel.app`

## 数据与安全

- 结构化访谈保存在浏览器本地的 `growthOSDiscoveryV3`
- 孩子画像、任务和进度默认保存在浏览器本地
- 不读取或转发ChatGPT Cookie、登录令牌
- Action通过Bearer API Key访问桥接后端
- 云端模型密钥只放在服务端环境变量
- 桥接请求7天自动过期
