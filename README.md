# 天赋任务OS

孩子自用的 AI 成长任务推荐 HTML5 app。

## 账户、角色与记忆

- 邮箱密码注册，密码使用 scrypt 哈希，会话使用 30 天 HttpOnly Cookie。
- 新账户注册时生成一枚20字符高熵恢复码，只向监护人显示一次；数据库仅保存独立盐值的scrypt哈希，不保存明文。
- 设置页可以在验证当前密码后轮换恢复码；本地测试管理员为了自动化验证可免输随机测试密码。
- 登录页支持使用邮箱、恢复码和新密码找回账户。重置成功会注销全部旧会话、让旧密码失效，并立即作废已使用恢复码。
- 恢复失败按IP与邮箱组合限制为15分钟最多5次；不存在账户也执行同成本哈希校验，减少账户枚举与暴力尝试风险。
- 成长档案提供“我的使用说明书”：GLM把任务体验、可共享作品、日记和行动记忆合并成“什么方法在什么场景可能有帮助”的个人策略。
- 每条策略必须引用至少两条真实来源；证据矛盾时降低置信度，禁止从一次经历推断稳定特点、人格或天赋。
- 孩子可以选择“对我有帮助 / 再观察 / 不适合我”。只有激活策略进入AI上下文；不确定和否定策略立即退出，并且重新生成不能靠换key复活。
- 删除记忆、日记或关闭作品AI权限会同步清除策略中的来源文本；剩余证据不足两条时策略自动降级并退出AI。
- 有可用策略后，任务推荐只携带最近8条原始记忆而非20条，并优先使用确认方法调整第一步与支持方式，形成Hermes式记忆压缩层。
- SQLite 持久化保存在 `data/growth-os.sqlite`，按账户和成长角色隔离。
- 可创建任意数量的成长角色；每个角色从起始路线出发，之后独立进化。
- 本地优先：浏览器保留离线副本，数据变化后自动同步数据库。
- 完成任务和复盘会形成可修正的长期证据记忆，AI 推荐会读取这些记忆，但不会据此给孩子贴固定标签。
- 每个角色可导出带版本号的 JSON，包含成长进度与记忆。
- 发现页会一次只唤醒一颗沉睡灵感，结合成长方向、孩子确认有效的方法、当前状态与安全消息生成新问题和10分钟尝试；收藏、稍后、放下都会改变未来推荐。
- 新建儿童角色必须记录监护人同意版本与时间；设置页可查看当前同意状态。
- 产品事件按账户和角色隔离，只记录必要行为属性，不保存孩子输入的自由文本。
- 设置页显示最近7天活跃天数、完整闭环和推荐接受率，用于验证产品是否真的有帮助。
- 执行页提供成长日记：支持自主记录、AI提问和二者结合；GLM会参考当天行动与近期日记提出一个温和问题，也能根据孩子尚未写完的草稿只追问一个值得继续探索的点。
- 日记草稿在AI思考和模式切换时保留；孩子可以随时直接保存，不需要完成一整轮对话。
- 日记独立存入数据库，保存时形成可撤回的第一人称记忆；删除日记会同步删除关联记忆。
- 每篇日记可单独选择是否允许进入AI上下文，私人保存的内容不会进入后续提问或任务推荐载荷。
- 可共享的日记标签、任务技能和复盘证据会形成“成长假设”，每条假设显示置信度、证据和反证。
- 孩子可以回答“像我 / 还不确定 / 不像我”；不确定或受挑战的假设会退出AI上下文，删除原始日记也会撤回相应证据。
- 本地测试环境默认自动登录 `admin@growth-os.local`，并创建“测试冒险家”角色；该免密入口在 `NODE_ENV=production` 时强制关闭，也可设置 `ENABLE_TEST_ADMIN=false` 关闭。
- 发现页提供“灵感工坊”：灵感按火花、孵化、行动、成果推进，GLM会生成澄清问题、三个不同方向和一个10分钟微行动。
- 日记可一键转成灵感；正在孵化或行动中的灵感会进入任务规划上下文，但不会垄断所有推荐。
- 首页提供自适应行动台：孩子可自行添加事项，系统按截止时间、重要度、当前能量、预计时长和进行状态选出一个“现在做”。
- 行动可由GLM拆成3-5个孩子可执行的小步和可观察完成标准；灵感进入行动状态时会自动生成10分钟行动。
- 自建行动完成后获得真实经验和宝石，撤销会同步撤回；每日行动经验最多60，避免通过切碎事项刷奖励。
- 首页“我的节奏”支持每天或上学日微习惯、触发提示、今日完成、主动暂停、撤销和连续节奏。
- AI今日指挥台先问能量、可用时间和孩子此刻的自主意图；“推进、创造、保持节奏、恢复”会真实改变候选排序，换一个或减轻任务时仍保留这次选择。
- “换一个”会轻问原因：太大、没兴趣、第一步不清楚或只是现在不合适。单次只改变当次选择，同一原因重复两次才校准以后推荐，并在界面解释AI正在适应什么。
- 成长档案可生成孩子先预览的家庭简报；只有孩子主动同意后家长设置页才显示，并可随时收回。后端只提供结构化行动证据、作品标题和已确认方法，绝不发送日记或私人问答正文。
- 同一行动或同名自主作品会形成真实版本轨迹；“做下一版”保留任务关联和作品名称，旧版本永不覆盖，作品架显示当前第几版和总版本数。
- 微习惯每次完成获得8经验、每天最多32；暂停不会被描述为失败。连续3/7/14/30等里程碑才沉淀长期成长证据，避免每天制造重复记忆。
- “现在做”可以进入专注舱，支持5/10/20分钟、暂停、继续、提前退出、只结束专注或同步完成行动。
- 专注会话保存在数据库，刷新页面可恢复；本周真实投入分钟会显示在行动台，但专注时长不直接奖励经验，也不用于评价孩子好坏。
- 成长档案提供“本周成长罗盘”：GLM只依据本周行动、习惯、专注、共享日记、灵感和可用成长假设总结证据，并给出一个下周微实验。
- 孩子可以反馈“像我 / 有点不像 / 下周试这个”；被否认的周报不会进入后续AI确认上下文，采用的微实验会写入本周成长计划。
- 私人日记不会进入周报证据；周报只转述共享内容，不展示私人原文，也不做诊断、比较或固定标签。
- 执行页把复盘绑定到具体任务；孩子用“难度、趣味、下次怎么帮”三个选择告诉系统真实体验，也可以只补一句话。
- 任务体验按角色存入数据库并随进度导出。一次反馈只作观察；至少两次同方向信号才调整挑战节奏，避免把当下状态误当成固定能力。
- 难度校准、近期任务体验、偏好的支持方式与经过重复验证的无聊玩法会进入AI推荐上下文；本地推荐失败回退时也使用同一套校准。
- 执行页提供“我的作品架”，支持把文字、照片、录音和项目链接绑定到已完成任务，也支持记录孩子主动完成的作品。
- 每件作品可单独控制AI参考权限；AI只接收作品名称、孩子说明和文字摘录，不读取照片或录音原文件。关闭权限或删除作品会撤回关联记忆与成长假设。
- 照片仅支持JPG/PNG/WebP，录音仅支持常见音频格式，单件不超过2MB；媒体读取要求当前账户权限并使用`nosniff`与私有缓存策略。
- 技能进度会参考作品证据且设置增量上限，不再只按任务完成次数机械增长；每周成长罗盘也会统计本周可共享作品。
- 首页默认使用“AI今日指挥台”：孩子只回答当前能量和可用时间，系统从自己的行动、今日习惯、正在孵化的灵感和恢复活动中选择一个下一步。
- 指挥台下方提供统一“随手记”：孩子只说一句，GLM会区分要做的事、灵感火花和成长感悟；意图不清楚时最多追问一次，孩子确认后才写入对应数据链路。
- 随手记不会把每个想法都任务化：灵感进入孵化池，感悟进入日记，只有明确行动才进入排序清单；行动仍保留时长、截止时间、第一小步和重复检测。
- 感悟在确认前可单独关闭后续AI参考权限，私人内容不会进入任务推荐、周报或策略压缩上下文。
- 任务复盘会询问“这次什么最让我愿意继续”：自主选择、看见进步、好奇结果、做出作品、分享连接、实际用途或还不知道。
- 动力回答是情境线索而非人格标签；同一线索至少重复两次才会调整每日指挥台的支持语，低能量时始终优先照顾状态。
- 动力线索进入周报、个人策略和数据导出（schema v8），孩子以后选择不同答案时会继续修正，而不是锁定一个固定类型。
- 路线页提供“我的成长方向”：孩子用一句愿望开始，GLM保留原意并整理成四周或三个月方向、可观察的进步信号和一个10分钟实验。
- 同时最多点亮三个方向；方向可以暂停、重新点亮、留下成果或删除，不使用成人KPI，也不把方向等同于天赋结论。
- 行动和灵感可关联成长方向；关联项进入每日候选时会获得适度加权并显示“正在推进什么”，但截止事项、能量和孩子的新兴趣仍可优先。
- 方向进度只来自真实证据：关联行动或灵感完成、关联行动留下作品；仅创建方向不会伪造进度。
- 删除方向只解除关联，不删除已有行动、灵感或作品。成长方向与关联证据已进入周报、任务生成上下文和数据导出（schema v8）。
- 成长档案首屏支持孩子主动提问，例如“我卡住时什么办法可能有用”；GLM只能根据服务器给出的可共享证据回答，并在回答下展示来源。
- 回答区分事实、观察和暂时线索；证据不足时明确说“还不知道”，不得诊断、比较、预测天赋或把一次经历说成固定规律。
- 私人日记不会进入问答候选。孩子自己的证据列表会显示“私人记录 / AI不参考”，避免“本机保存”和“AI可读取”混淆。
- 孩子可反馈“这对我有帮助 / 这不像我”或删除整次问答；否定过的回答会进入下一次提问的纠错上下文。问答历史包含在 schema v8 数据导出中。
- 行动台支持“现在不合适”任务协商：孩子先选择没力气、没时间、不清楚或不重要，GLM再给一个不带羞辱和损失恐惧的调整建议。
- 协商结果包括缩成5分钟、明天再做、放到以后和不做了；延期或放下不会扣经验，也不会被解释成懒惰或失败。
- 明天、以后与已放下的行动不会进入今日排序或每日指挥台；它们保留在可展开的“以后再看”区域，孩子可随时重新加入。
- 每次决定会保存原因与结果，用于避免反复推荐同一不合适任务，并包含在 schema v8 导出中；删除行动会同步删除它的决定历史。
- 协商历史会形成可撤回的“当前节奏校准”：同一原因至少重复两次才生效，一次延期不会被当成规律。
- 重复“没时间”时优先短任务，重复“没力气”时优先低能量任务，重复“不清楚”时优先已有第一步的任务，重复“不重要”时提高重要事项权重。
- 行动台会说明AI正在适应哪种当前节奏；每日指挥台使用校准时显示“节奏适配”，避免个性化成为黑箱。
- 当前节奏不是能力、性格或固定偏好。删除相关行动和决定后样本会减少，低于门槛时校准与提示同步退出。
- GLM只能复制候选事项的引用，队列不为空时禁止创造新任务；模型不可用时使用相同约束的本地评分回退。
- “换一个”会把当前事项加入当天排除集，避免只改措辞后重复推荐；“轻一点”会把状态改为低能量、可用时间缩为5分钟后重新选择。
- 点击“现在开始”会把已有行动切换为进行中并直接进入专注舱。完整行动台、习惯管理和探索型新任务改为孩子主动展开，默认首页不再同时展示多套清单。
- 完整行动台使用“AI收件箱”替代时间、重要程度、能量、截止日期四字段表单。孩子说一句自然语言，AI先整理为草稿，不会直接创建。
- 信息不足时最多追问一个可点选问题；回答后禁止继续追问。GLM超过10秒会立即使用本地日期与时长解析回退，记录想法不被模型延迟阻塞。
- 草稿会展示标题、预计时长、能量、重要程度、截止时间和第一小步，孩子确认后才进入行动队列。
- 新草稿会与所有未完成行动做标题和语义相似度检查；发现重复时打开原行动，不再制造第二份待办。

这是向 Hermes 式“可持续进化系统”迈进的持久化基础。正式对外部署前还需补充记忆合并、技能图版本、家长同意、账户找回、数据库加密与权限审计。

产品决策和技能树依据见 [SCIENCE.md](SCIENCE.md)。

```bash
npm start
```

打开：

```text
http://localhost:5173
```

## SiliconFlow 配置

不要把 API Key 写进 `index.html` 或 `app.js`。

在本机创建 `.env.local`，内容格式参考：

```bash
SILICONFLOW_API_KEY=your_key_here
SILICONFLOW_BASE_URL=https://api.siliconflow.cn/v1
SILICONFLOW_MODEL=zai-org/GLM-5.2
PORT=5173
```

如果 SiliconFlow 返回 model not found，只需要把 `SILICONFLOW_MODEL` 改成控制台显示的准确模型名。

## 大模型规划链

任务不是只从固定清单里抽取。前端会把以下上下文组合成结构化 JSON：

- 孩子的八维技能树、长期画像和当天回答
- 最近完成、复盘、推荐历史和明确跳过的任务
- 本周成长计划与模型生成的里程碑
- 近期日程、预计能量消耗和家庭限制
- 家长手动加入或自动同步的安全新闻消息
- 孩子未否认的最近一份成长罗盘，以及已主动采用的下周微实验

`GLM-5.2` 生成任务后，服务器会计算它与近期任务的相似度；过于相似时自动再生成一次，并选择差异更大的结果。

## API

前端调用：

```text
POST /api/coach
POST /api/plan
GET  /api/task-feedback?profileId=...
POST /api/task-feedback
GET  /api/artifacts?profileId=...
POST /api/artifacts
PATCH/DELETE /api/artifacts/:id
GET  /api/daily-plan?profileId=...
POST /api/daily-plan/generate
POST /api/action-inbox/parse
POST /api/auth/recovery/rotate
POST /api/auth/recovery/reset
GET  /api/strategy-insights?profileId=...
POST /api/strategy-insights/generate
PATCH /api/strategy-insights/:id
GET  /api/status
GET  /api/context/news?topics=AI教育,科学发现
```

后端会代理到：

```text
POST {SILICONFLOW_BASE_URL}/chat/completions
```

模型必须返回 JSON，前端会把 `recommendation` 渲染成一个孩子可执行的任务。

新闻上下文优先使用 [GDELT DOC 2.0](https://blog.gdeltproject.org/gdelt-doc-2-0-api-debuts/)，访问缓慢时回退到 [HN Search by Algolia](https://hn.algolia.com/api)。标题会经过儿童安全与主题相关性过滤，新闻只作为项目灵感，不直接展示成人化内容。

## 经验与宝石

- 每个任务明确给出经验值。
- 完成后按约 `6 XP = 1 宝石` 入账。
- 当前经验达到等级上限时自动升级，下一等级门槛增加约 15%。
- 撤销完成会同步撤销经验和宝石。
- 顶部宝石、底部等级、经验条和设置内账本都读取同一份本地完成记录。
