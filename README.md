# 成长OS

成长OS不是一个让家长先选技能、再拼任务的配置工具，而是一套“通过对话理解孩子，再用真实项目验证判断”的成长系统。

## 当前版本：V5.0 对话发现

新的主流程：

1. 聊孩子
2. 形成发现
3. 映射未来技能路线
4. 自动推荐项目
5. 执行、留证据并反向修正画像

## 已部署地址

- GitHub Pages：`https://sddzcuigc.github.io/growth-os/`
- Vercel：`https://growth-os-ten-pearl.vercel.app`

## 一、聊孩子

系统不再要求家长一开始就选择“项目外壳”和“核心技能”。

它会一次询问一个维度：

- 真正会主动投入的事情
- 遇到不会的问题时怎样处理
- 学校中的真实表现
- 启动、持续、检查和收尾
- 遇挫后的反应与恢复
- 更自然的表达方式
- 合作、边界和同伴关系
- 体力、睡眠、姿态与精神状态
- 家庭资源和时间限制
- 半年内最希望发生的变化

用户可以自由回答、暂时跳过，也可以随时补充或纠正。

## 二、形成发现

系统将对话整理为四类信息：

- 已观察到的事实
- 可能的优势或天赋苗头
- 当前最值得补强的环节
- 仍然缺少、需要继续观察的信息

每条判断必须显示：

- 依据来自哪段回答
- 当前置信度
- 对未来培养意味着什么

这些结果是“待验证假设”，不是永久标签。

## 三、未来技能路线

系统自动把发现映射到成长OS技能树，不再要求家长自己判断应该选什么能力。

路线分为四层：

1. 利用优势：让孩子愿意进入项目
2. 核心补强：本阶段只抓1—3项关键能力
3. 重要支撑：在真实项目中自然发生
4. 基础维护：每个项目都要检查的底层能力

未来能力方向包括：

- AI与软件产品
- 科学与数据
- 工程与系统
- 内容与传播
- 商业与组织
- 公共服务与领导

## 四、自动推荐项目

系统根据以下信息自动推荐三个项目：

- 孩子真正愿意投入的兴趣线索
- 当前短板
- 未来技能树
- 家庭资源
- 可用时间
- 真实使用者

每个推荐会解释：

- 利用了孩子什么优势
- 重点补什么短板
- 对应哪些未来技能
- 为什么现在适合做

家长只需要判断哪个项目最自然，不需要先成为教育专家。

仍保留“高级调整”，用于手动修改项目外壳、核心技能、真实使用者和时间。

## 五、执行与反向验证

项目不是课程表，而是一次诊断实验。

执行过程中记录：

- 是否需要提醒
- 卡在哪一步
- 是否能独立恢复
- 是否真正完成检查和收尾
- 真实使用者的反馈
- 是否根据反馈修改
- 能否把方法迁移到新场景

这些证据会反过来修正孩子画像和技能优先级。

## 当前对话能力说明

V5.0已经实现对话式、逐题、可补充的引导流程，并在浏览器本地完成第一轮证据提取、优势线索、短板识别和技能映射。

当前这部分不是自由发挥的大模型聊天，而是可解释的引导式访谈内核。这样即使没有配置API，也能完整运行，并且每个结论可以追溯到具体回答。

后续可将访谈阶段接入多供应商API或自定义GPT，让模型根据上一条回答动态生成更深的追问；本地内核仍作为失败回退和可解释基线。

## Plus Action自动桥接

Plus Action继续用于深度项目规划：

1. 成长OS创建规划请求
2. 自动打开用户自己的自定义GPT
3. 用户发送一次任务编号
4. 自定义GPT读取孩子发现、技能树和项目建议
5. GPT生成完整项目并写回
6. 成长OS轮询、导入并进入执行

成长OS不能代替用户在ChatGPT界面自动发送消息，因此仍需用户发送一次任务命令。

## Vercel桥接环境变量

必须配置：

- `GROWTH_OS_BRIDGE_KEY`
- Redis REST地址与Token，支持以下任一组：
  - `UPSTASH_REDIS_REST_URL` + `UPSTASH_REDIS_REST_TOKEN`
  - `KV_REST_API_URL` + `KV_REST_API_TOKEN`
  - `REDIS_REST_URL` + `REDIS_REST_TOKEN`
- `ALLOWED_ORIGINS`：建议填写 `https://sddzcuigc.github.io,https://growth-os-ten-pearl.vercel.app`

## 自定义GPT Action

仓库包含：

- `custom-gpt-instructions.md`
- `gpt-action-openapi.yaml`

Action认证：

- Authentication：API Key
- Auth Type：Bearer
- API Key：与 `GROWTH_OS_BRIDGE_KEY` 完全相同

主要接口：

- `GET /api/bridge/health`
- `POST /api/bridge/create`
- `GET /api/bridge/get?id=...`
- `GET /api/bridge/latest`
- `POST /api/bridge/result`
- `GET /api/bridge/status?id=...`

## 数据与安全

- 孩子对话、画像、任务和进度默认保存在浏览器本地
- 对话发现单独保存在 `growthOSDiscoveryV2`
- 不读取或转发ChatGPT Cookie、登录令牌
- Action通过Bearer API Key访问桥接后端
- 云端模型密钥只放在服务端环境变量
- 桥接请求7天自动过期
