# 成长OS

成长OS是一套“通过多情境证据理解孩子，再用真实项目验证判断”的双孩成长系统。

## 当前版本：V5.2 双孩AI语音

主流程：

1. 选择孩子
2. 用语音或多选记录真实行为
3. AI把自然描述整理成结构化观察
4. 形成阶段性发现
5. 映射未来技能路线
6. 自动策划项目
7. 执行、留证据并反向修正画像

## AI语音输入

每个访谈主题都有“AI语音观察”区域。

使用流程：

1. 点击麦克风
2. 用自然语言连续描述孩子
3. 点击结束
4. 点击“AI整理并填入”
5. AI自动整理：
   - 多个行为选项
   - 出现频率
   - 发生情境
   - 具体事例
   - 不确定性说明
   - 一个最有价值的追问
6. 家长人工确认后，再保存本次观察

示例语音：

> 上周做数学作业时，他前二十分钟自己完成，遇到应用题后马上喊不会。我没有讲答案，只问他卡在哪一步，他停了两分钟，又自己画图做出来了。但做完以后没有检查。

系统会同时保留“能独立启动”“遇难题容易求助”“在提示后能恢复”“缺少检查”等多个信息，不会压成单选结论。

### 语音与AI的两层处理

第一层是浏览器语音转文字：

- 在支持Web Speech API的浏览器中，点击麦克风即可连续听写
- 建议使用最新版Android Chrome
- 不支持网页语音识别时，可以直接使用手机输入法麦克风

第二层是AI结构化整理：

- 文字只有在用户点击“AI整理并填入”后才发送到配置的AI后端
- AI只能从原话和当前选项中提取，不得编造
- AI可以同时选择多个互相矛盾的行为
- AI结果必须由家长确认后保存
- 云端AI不可用时，会回退为本地关键词初步整理，并明确标注不是大模型结果

## AI语音后端

接口：

```text
POST /api/observe
```

复用成长OS多供应商配置，支持：

- OpenAI
- DeepSeek
- Kimi / Moonshot
- 智谱GLM
- GPUStack
- 其他OpenAI兼容API

Vercel需要配置：

- `GROWTH_OS_ACCESS_CODE`
- 对应供应商API密钥，例如 `OPENAI_API_KEY`
- 可选模型变量，例如 `OPENAI_MODEL`
- `ALLOWED_ORIGINS`

前端在“项目 → AI内核设置”中填写：

- 运行方式：多供应商API
- 统一后端地址：`https://growth-os-ten-pearl.vercel.app/api/compile`
- 个人访问码：与 `GROWTH_OS_ACCESS_CODE` 相同

语音整理会自动把 `/api/compile` 转换为 `/api/observe`。

## 结构化访谈

每个主题由五部分组成：

- 多选行为观察
- 出现频率
- 主要情境，可多选
- 信息来源
- 具体事例与补充说明

九个主题：

1. 自主兴趣与内在动机
2. 学习、理解与解决问题方式
3. 任务执行与自我管理
4. 表达、创造与成果呈现
5. 情绪、挫折与恢复
6. 合作、边界与同伴关系
7. 身体状态与生活节律
8. 家庭环境、资源与现实限制
9. 半年目标与优先级

信息来源可以是家长、孩子、老师、老人、教练或兴趣老师。

## 发现报告

报告分开呈现：

- 已明确观察到的事实
- 优势与天赋苗头
- 当前最值得支持的能力
- 矛盾与情境差异
- 仍然不知道的信息

“天赋”不会由一次勾选确认。只有当优势跨时间、跨情境、低催促地重复出现，并能形成真实成果时，置信度才会上升。

## 项目与反向验证

项目不是课程表，而是验证假设的实验。执行中记录：

- 哪些情境表现更好
- 成人介入多少
- 卡点发生在哪一环
- 是否能自己恢复
- 是否检查和收尾
- 真实使用者如何反馈
- 是否根据反馈修改
- 能否迁移到新场景

这些证据会反过来修正孩子画像和下一轮技能优先级。

## 数据与安全

- 双孩数据保存在浏览器本地的 `growthOSFamilyV1`
- 当前孩子访谈兼容保存在 `growthOSDiscoveryV3`
- 首次升级前的单孩数据备份在 `growthOSLegacyBeforeFamily`
- 不读取或转发ChatGPT Cookie、登录令牌
- 云端AI密钥只放在Vercel环境变量中
- 语音转写文字只有在用户主动点击AI整理后才发送
- 不建议录入不必要的身份、医疗或高度敏感信息
