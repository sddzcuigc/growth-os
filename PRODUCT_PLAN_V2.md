# 成长OS V2：证据驱动的儿童个人成长系统

更新：2026-07-15

## 1. 产品定义

成长OS不是传统Todo、AI家教或天赋测评。它是给孩子自己使用、由监护人提供边界的个人成长系统：AI通过一次一个问题认识孩子；孩子确认或修正画像；系统把画像与面向2030以后的能力框架结合，形成成长蓝图、SMART目标、OKR和每天一个下一步；日记、行动、作品与真实体验成为新证据，再更新后续蓝图。

核心承诺：孩子越来越懂自己、越来越能自己行动；家长越来越少靠催促；AI越来越会提供恰当的脚手架，但不替孩子思考和完成。

## 2. 唯一主工作流

```text
注册/新建角色
  → AI一次问一个问题
  → 生成可修正画像
  → 孩子确认“像不像我”
  → 未来能力树匹配
  → 两项成长优先级（一个底座 + 一个探索）
  → 四周SMART目标与OKR
  → AI指出当前一步 + 今日多领域任务册
  → 行动/卡住/换选/日记/作品/复盘
  → 新证据进入画像与蓝图
  → 孩子再次确认变化
```

底部五页承担连续工作，而非五个工具箱：

| 页面 | 只回答一个问题 | 在闭环中的职责 |
| --- | --- | --- |
| 今天 | 我现在做什么 | 状态问答、当前一步，以及健康/学习/生活/未来能力的完整微任务册 |
| 灵感 | 我还想探索什么 | 保存兴趣，不强制变成待办 |
| 能力 | 我为什么练这些 | 可修正画像、成长蓝图、SMART/OKR、技能证据 |
| 计划 | 接下来怎样推进 | 周里程碑、日程约束、行动队列 |
| 记录 | AI从什么学到我 | 日记、体验反馈、作品、复盘、撤回证据 |

## 3. V2未来能力树

能力树不是职业预测，也不声称能准确预言孩子未来。它把较稳定的学习科学与快速变化的技术趋势分开：每年复审趋势层，底座层只有在证据充分时调整。

| 能力 | 儿童可理解定义 | 可观察证据 |
| --- | --- | --- |
| 身心底座 | 我会按身体和情绪状态调整 | 睡眠/运动记录、主动恢复、合理暂停后补做 |
| 自我调节 | 我能开始、坚持、检查和收尾 | 减少提醒、完成闭环、忘记后主动补救 |
| 会学会想 | 我知道哪里不会并会换办法 | 预测、监控、错误发现、策略迁移 |
| 表达沟通 | 我能讲清意图、判断和作品 | 讲解、写作、提问、根据听众反馈修改 |
| 数据推理 | 我用数量、模式和证据判断 | 预算、比较、记录、核验结论 |
| 创造项目 | 我把问题变成作品并迭代 | 最小版本、测试、真实成果、版本变化 |
| AI协作 | 我理解和使用AI但保留人的决定 | 自己先想、AI查漏、来源核验、人机分工说明 |
| 判断协作 | 我考虑公平、责任、来源和分工 | 轮换角色、来源说明、影响检查、合作复盘 |

选择规则：任何时刻最多点亮两个蓝图优先项。第一项优先补足支持所有学习的底座，第二项连接孩子当下兴趣与未来能力。学校基础、真实生活、运动与责任不能被AI项目挤掉。

## 4. 研究依据与产品解释

- [WEF Future of Jobs Report 2025](https://www.weforum.org/publications/the-future-of-jobs-report-2025/in-full/3-skills-outlook/)指出AI与大数据、技术素养增长很快，同时创造性思维、韧性、灵活性和协作仍是核心。产品因此不做单一“编程技能树”。
- [OECD Learning Compass 2030](https://www.oecd.org/en/data/tools/oecd-learning-compass-2030.html)以学生主体性、福祉及知识/技能/态度/价值为核心；其行动逻辑是预期、行动、反思。产品将此落成孩子确认画像、自己选择、行动和复盘。
- [UNESCO AI Competency Framework for Students](https://www.unesco.org/en/articles/ai-competency-framework-students)包含以人为本、AI伦理、技术应用和系统设计，并按理解、应用、创造进阶。产品要求“自己先想、AI提问/查漏、孩子核验并说明分工”。
- 2024年的[元认知校准研究](https://www.nature.com/articles/s41539-024-00231-z)支持把自我监控、错误发现与策略调整视为自我调节学习的重要部分；产品不只记录完成，还记录预测、卡点和复盘。
- 2024年的[预注册学习策略实验](https://www.nature.com/articles/s41539-024-00245-7)显示，反驳错误信念与元认知提示能改善策略选择；产品应通过提问和可见反馈教方法，而非只派任务。
- [How People Learn II](https://nap.nationalacademies.org/catalog/24783/how-people-learn-ii-learners-contexts-and-cultures)强调学习者、情境、文化、动机和发展阶段的共同作用；产品必须读取家庭资源、健康约束和真实日程。

研究边界：WEF是雇主调查，不是儿童课程处方；单项实验不能直接证明App有效；所有能力权重都要通过真实家庭使用、儿童反馈和纵向证据继续验证。

## 5. 动态画像与证据模型

### 三层真相

1. 孩子确认的自我描述：最高优先级，AI不能覆盖。
2. 可修正成长假设：来自重复证据，显示来源、置信度、反证和更新时间。
3. 当天状态：只影响今天，不升级为人格或能力标签。

### 证据权重

- 高：孩子明确更正；连续多次独立行为；真实作品的版本变化。
- 中：重复任务体验；多个场景中出现的同一策略；孩子确认的周复盘。
- 低：单篇日记；一次换选；家长单次观察；模型推断。

日记默认由孩子决定是否进入AI上下文。删除日记、作品或记忆时，相关假设、蓝图证据和奖励都应可撤回。健康信息只作为任务安全约束，不用于诊断或排名。

## 6. 个性化规划引擎

输入：已确认画像、年龄、家庭约束、未来能力树、活跃目标、日程、日记、任务体验、卡点、完成/撤回、作品、周复盘、经过筛选的新闻灵感。

输出顺序：

1. 更新证据摘要和置信度，不直接改写孩子画像。
2. 选一个底座能力和一个探索能力，解释“为什么是现在”。
3. 每项生成一个四周SMART Objective。
4. 每个Objective最多3个可观察Key Result。
5. 生成本周里程碑和今天10分钟内可开始的一步。
6. 首页同时提供约30到40项多领域微任务，按基础、成长、探索分层；AI只负责指出此刻优先项，不把其余任务隐藏。
7. 微任务按年龄、兴趣、蓝图重点和日期轮换表达，覆盖健康、学习、生活技能、责任、表达、AI与创造。
8. 每日推荐从已有目标行动、习惯、灵感、任务册和恢复动作中选一个；没有可执行项时才创建一个蓝图实验。
9. 完成后只问必要反馈：难度、兴趣、帮助方式、动力和一句发现。

### 关卡难度规则

任务不是按日期机械变难。每个能力分别计算1到5关：角色总等级和对应技能证据推动升阶；作品、独立完成和“太简单”反馈会提高挑战；低能量、连续卡住或年龄限制会临时降阶。五级依次为：接受脚手架完成、尽量独立、独立并检查、发现问题并迭代、迁移到新情境或教给别人。

每项任务必须显示实际来源，最多四项：孩子确认画像、AI成长蓝图、SMART目标、近期日记、历史任务体验、今日状态、家庭成长档案、未来技能树或学习科学知识库。完成证据同时保存关卡、难度、成功标准和来源，供后续画像与计划调整。

模型失败、超时或限流时使用本地规则生成可解释蓝图；任何模型输出必须通过固定Schema、年龄、安全、时长、重复度和证据引用校验。

## 7. 首个家庭的差异化策略

哥哥的初始假设：用故事、知识、地图、Minecraft等兴趣作为入口，优先发展身心/自我调节底座和“兴趣到作品”的闭环；任务需要先示范方法、给清楚第一步，逐步撤掉帮助。睡眠、呼吸、哮喘和姿态只作为状态约束，不作能力解释。

妹妹的初始假设：保留稳定执行优势，但不因“更能干”增加责任；优先提供选择、提问、创造和允许不完美的任务，观察她真正愿意持续的兴趣。兄妹合作必须轮换主持和检查角色，避免固定成“哥哥指挥、妹妹执行”。

家庭执行形态：少量固定基础项 + 每周一个技能主题 + 老人只负责看懂和简单执行 + 父母晚间10到20分钟复盘 + 高风险运动/烹饪由可靠成人现场负责。计划需要可打印、可打勾、少文字。

## 8. 游戏化边界

经验代表真实投入，技能经验代表对应证据，宝石用于可见但非竞争性的外观/选择权。基础奖励固定且事先可理解；小概率彩蛋有每日上限、不能付费抽取、不能因中断连续天数羞辱孩子。禁止排行榜、无限连胜压力、付费随机箱、损失厌恶文案和故意拖延反馈。

成功不等于“做得多”，而是独立性提高、方法迁移、能主动补救、作品迭代和对自己的理解更准确。

## 9. 数据与系统架构

- 账户：注册、登录、找回、监护人同意、多角色。
- 数据库：画像快照、记忆、假设、日记、任务体验、作品、目标、行动、习惯、专注、周报、成长蓝图、事件。
- 成长蓝图：保存输入证据指纹、模型/本地来源、两项优先能力、四周路线与更新时间。
- AI：SiliconFlow OpenAI兼容接口，默认`zai-org/GLM-5.2`；密钥只放服务端环境变量，绝不进入浏览器或仓库。
- 导出：按角色导出可读JSON，后续补充打印版成长档案和家庭迁移包。
- 隐私：日记与作品逐条授权；数据删除可级联撤回；家长简报不读取私人日记原文。

## 10. 迭代路线

### Sprint 1：闭环统一（当前）

- 持久化成长蓝图V2。
- 蓝图读取画像、日记、体验、作品、目标和行动。
- 蓝图进入教练生成上下文。
- 新证据自动标记并合并更新。
- 能力页展示证据、置信度、SMART/OKR和第一步。

### Sprint 2：蓝图到每日行动

- 当目标没有开放行动时，自动把蓝图第一实验转成一条行动。
- 每日推荐显示“它推进哪个Objective/KR”。
- 孩子可回答蓝图的下一问题，并直接纠正某一优先项。
- 增加蓝图版本对比：保留、增强、减弱、未知。

### Sprint 3：家长与真实生活

- 家长单独空间、少催促建议和健康/安全约束。
- 可打印周卡与老人模式。
- 家庭资源盘点、日历接入、真实项目模板。
- 兄妹协作的角色轮换和公平检查。

### Sprint 4：质量与规模

- 建立匿名化推荐评估集：年龄、安全、个性化、重复度、可执行性、证据真实性。
- 模型成本、延迟、回退率和Schema失败看板。
- 账户找回、数据库加密、权限审计、删除与导出验收。
- PWA离线、通知、付费与家庭订阅。

## 11. 指标与验收

北极星指标：每周由孩子独立完成至少一次“提问 → 选择 → 行动 → 证据 → 复盘 → 蓝图更新”的家庭数。

- 激活：注册后24小时内确认画像并开始第一个目标。
- 易用性：新用户不经说明能在3分钟内找到今天唯一下一步。
- 个性化：推荐能引用至少一条真实上下文；连续三次高度同质任务低于10%。
- 成长：四周内独立开始率、闭环率、策略迁移和作品版本数上升。
- 主体性：孩子纠正画像/蓝图的入口100%可见；纠正后旧结论不再参与推荐。
- 安全：诊断、羞辱、成人焦虑、付费随机奖励和不适龄高风险任务进入率为0。
- 可靠性：核心流程E2E全绿；模型失败时本地兜底可完成主流程。

## 12. 当前不宣称

不宣称发现固定天赋，不预测职业，不诊断心理或健康问题，不承诺提高分数，不把相关性说成因果。产品当前是家庭MVP，需要至少20个家庭、4到8周真实使用与儿童访谈后，才能校准能力权重和商业假设。
