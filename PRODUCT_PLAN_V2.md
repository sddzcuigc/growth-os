# 成长OS V4：成长底座 + 项目主线驱动的儿童成长系统

更新：2026-07-16

> 文件名暂时保留为 `PRODUCT_PLAN_V2.md`，本文内容与产品版本已升级为 V4。
> 本文是产品摘要。涉及画像、技能树、蓝图、目标、三轨任务、日程、番茄钟、Boss、证据、奖励和验收的实现冲突时，以 `成长OS_100Boss每日小Boss与每周Boss战_完整设计_V1.0.md` 为唯一主规范。

## 1. 产品定义

成长OS不是传统 Todo、AI 家教、天赋测评，也不是五个互不相关的成长工具。它是给孩子自己使用、由监护人提供安全边界的个人成长系统。

系统通过 AI 一次问一个问题认识孩子，由孩子确认或修正画像；再把画像、未来能力框架、真实生活条件与近期证据汇成一条“当前成长主线”。这条主线向上连接 SMART 目标和 OKR，向下生成每日任务册；孩子的完成情况、日记、作品和真实体验再回流，动态调整画像、蓝图、路线与下一份任务册。

V4 修正了 V3 的关键误区：**一条项目主线不等于孩子每天只练一种能力。** 阅读、基础学习、身体活动、卫生、自理、责任、交流和兴趣特长是长期成长底座，不能因为本周项目主题不同而被 AI 删除。项目主线负责深度和成果，成长底座负责广度、健康和稳定节奏。

核心承诺：

- 孩子越来越懂自己，也越来越能独立开始、调整和完成事情。
- 孩子每天减少选择负担，只需看见适合今天的下一关。
- 家长越来越少依赖催促，更多看见孩子的策略和真实进步。
- AI 提供恰当脚手架，但不替孩子判断、思考或完成。

## 2. V4 不可妥协的系统结构

### 2.1 每日三层任务，而不是一张混合清单

| 层级 | 作用 | 来源 | 是否由 AI 自由生成 | 是否影响 Boss |
| --- | --- | --- | --- | --- |
| 固定成长底座 | 保持健康、学习和生活的基本节奏 | 家庭确认的例行库、年龄与安全规则 | 否；AI 只能调时间、难度和替代项 | 否 |
| 本周项目任务 | 推进当前 SMART/OKR 和周成果 | 画像、蓝图、目标、项目契约和日程 | 是，但必须受项目阶段约束 | 是 |
| 今日加餐 | 兴趣探索、技能补弱和偶发机会 | 日记、灵感、新闻、体验和技能树 | 是；孩子自主选择 | 否 |

固定成长底座默认覆盖：

- 身体：喝水、早餐、户外活动、跑跳、跳绳、球类、拉伸、视力休息和睡眠准备。
- 卫生与自理：起床、刷牙、洗脸、衣物、书包、书桌和个人物品归位。
- 基础学习：学校任务检查、回忆练习、中文、数学、英文和学习准备。
- 阅读：每天自由阅读，并用一句话、图或讲述留下理解证据。
- 生活责任：家务、备餐、物品维护、发现错误后补救和完成后收尾。
- 交流能力：倾听、复述、表达感受、提出问题、求助、分享和协作。
- 特长发展：由孩子和家庭选定 1 至 3 个长期方向，按周频率出现，不因单周项目消失。

例行库可以包含数十到上百种轻活动，但“今天”只展开家庭当前启用并适合当日状态的项目。固定不等于僵硬：生病、疲劳、天气、课程和家庭事件可以触发暂停、缩小或替换，不能触发羞辱、断签或扣除已获得奖励。

### 2.2 一周一个主线项目

每周 Boss 必须绑定一个 `weeklyProject`，它不是能力名称的包装，而是可执行的项目契约：

```text
weeklyProject
  ├─ drivingQuestion：孩子能理解的驱动问题
  ├─ weeklyProduct：周末可展示、可使用或可讲清的成果
  ├─ targetKnowledge：需要学习的知识
  ├─ targetSkills：重点能力与关联基础能力
  ├─ audience：成果给谁看或为谁使用
  ├─ successCriteria：3 条可观察标准
  ├─ dailyStages：定义 → 资料 → 方案 → 初版 → 测试 → 展示 → 复盘
  └─ evidence：阅读记录、草图、初版、反馈、改版和最终成果
```

Boss 项目化判定必须同时满足：

1. 有一个真实问题或明确成果，而不是“练习沟通能力”之类抽象口号。
2. 连续多天推进同一成果，日任务之间存在依赖关系。
3. 至少包含一次资料阅读或观察、一次制作、一次反馈和一次改进。
4. 最终有公开给可信任对象的产品、讲解、演示或真实使用。
5. 周 Boss 的伤害来自项目证据质量，不能只靠点击任务或填写一句套话。

### 2.3 每日 Boss 只验证项目阶段

每日小 Boss 对应当天项目阶段：周一明确问题，周二寻找资料，周三设计方案，周四完成初版，周五测试改进，周六展示成果，周日复盘迁移。卫生、刷牙和阅读等固定底座会获得自己的经验，但不会成为击败 Boss 的强制门槛，避免把基本照顾变成交易。

### 2.4 日程是约束，不是装饰

- 固定例行项目带有时段、频率、预计时长和可替代项。
- 学校课程、培训、家庭活动和休息以只读日程进入规划器。
- 项目任务必须放进真实可用时间，不能与课程重叠，也不能把全天空白误判为无限时间。
- “今天”采用时间顺序展示；“本周”同时展示固定例行、项目里程碑和外部日程。

## 3. 九个大功能必须服务同一闭环

| 功能 | 唯一职责 | 必须产出 | 下游使用者 |
| --- | --- | --- | --- |
| 画像 | 描述当前状态、兴趣、支持方式和不确定性 | 孩子可更正的画像版本 | 技能树、蓝图、任务生成 |
| 技能树 | 展示成长底座、未来能力和特长方向 | 能力节点、证据、熟练阶段 | 蓝图、项目选题、难度 |
| 蓝图 | 决定四周优先级和能力组合 | 1 个底座重点、1 个探索重点、1 条主线 | SMART/OKR、周项目 |
| 灵感 | 收集好奇、问题和兴趣信号 | 可保留、孵化或用于下轮选题的信号 | 画像、蓝图、项目题库 |
| 日志 | 保存感悟、困难、方法与自主叙事 | 私密或授权的第一人称证据 | 画像校准、反思问题 |
| Todo | 执行固定底座、项目任务和今日加餐 | 完成、调整、延期和跳过证据 | 经验、画像、周复盘 |
| 番茄钟 | 帮助开始和保持专注 | 与具体任务绑定的专注记录 | 难度校准、执行证据 |
| Boss | 把周项目变成有连续性的挑战 | 每日阶段证据、周成果、符文和复盘 | 技能证据、奖励、下一周 |
| 日程 | 保护真实时间和家庭约束 | 可用时间块、冲突与提醒 | Todo 排序、项目计划 |

任何功能若不能读入上游上下文或写出下游证据，就是孤岛，不能通过验收。

## 4. 交互与产品基准

- Todoist：学习重复任务与日历事件分离、Today/Upcoming 的时间视图；成长 OS 必须区分例行、任务和外部事件。
- Habitica：学习 Habits、Dailies、To-dos 的任务分型；不采用掉血、惩罚性断签和压力式社交竞争。
- Finch：学习按领域组织大量目标、首页显示今日目标和温和的小步支持。
- Khan Academy：技能按掌握而非单次完成升级；同一能力需要跨任务和跨场景证据。
- Duolingo：学习降低每日开始门槛和明确进度反馈；不把连续天数当成道德评价。
- PBLWorks：项目必须有驱动问题、持续探究、真实性、孩子选择、反思、反馈改版和公开成果。

## 5. V4 验收门槛

- 基础覆盖：启用默认家庭模板后，每天可见卫生、身体、学习、阅读、责任和交流六类，不允许被 AI 覆盖为空。
- 项目一致：周 Boss、每日小 Boss、主线 Todo 和周成果使用同一个 `weeklyProjectId`。
- 项目质量：每周项目包含驱动问题、成果、受众、成功标准、七日阶段、反馈改版和最终证据。
- 来源清楚：每项任务显示“固定例行 / 主线项目 / 今日加餐”，并能追溯到模板、目标或证据。
- 日程可行：任务总时长不得超过可用时间；冲突时先调整，不向孩子施压。
- 功能闭环：画像、技能树、蓝图、灵感、日志、Todo、番茄钟、Boss 和日程均有至少一条自动化端到端测试。
- 儿童可用：孩子不看教程，在 30 秒内能说出“现在该做什么、为什么、做完算什么”。
- 家长可控：固定例行和特长方向由家庭确认，AI 不能擅自增加医疗、危险运动或高压学业要求。

下面保留 V3 的单一主线设计，并将其视为 V4 的“项目主线层”。它不再拥有删除成长底座的权力。

## 6. V3 核心保留：只有一条当前项目主线

V2 的主要问题不是功能不足，而是缺少统一中心：画像、蓝图、目标、周计划、推荐任务、灵感和日记可以分别工作，却没有持续服务于同一个目的。

V3 以“当前成长主线 `journeyId`”作为全系统中心对象：

```text
当前成长主线
  ├─ 已确认画像：我现在是谁、喜欢什么、怎样更容易行动
  ├─ 成长蓝图：当前最值得发展的底座能力与探索能力
  ├─ SMART Objective：这一轮具体要发生什么变化
  ├─ OKR：最多3个可观察关键结果
  ├─ 四周路线：本周里程碑、现实约束与日程
  ├─ 今日任务册：AI从统一上下文生成的多领域微任务
  └─ 成长证据：完成、反馈、日记、灵感、作品和复盘
```

产品规则：

1. 同一角色同一时刻只有一条“当前主线”。
2. 可以保留暂停、完成或备用方向，但它们不生成今日任务。
3. 新建或重新点亮方向时，它成为新的当前主线，原内容不会丢失。
4. 日记、作品、任务体验、灵感和行动默认记录 `journeyId`。
5. 删除主线只解除关联，不删除孩子已经留下的日记、作品和成长证据。
6. 今日当前任务必须来自同一份 AI 今日任务册，不能再由另一套推荐逻辑生成。

## 3. 唯一主工作流

```text
注册 / 新建角色
  → AI一次问一个问题
  → 生成可修正的孩子画像
  → 孩子确认“像不像我”并更正
  → 匹配未来能力树与真实兴趣
  → 形成一条当前成长主线
  → 建立SMART Objective与最多3个KR
  → 编排四周路线与本周里程碑
  → AI生成今日多领域任务册
  → AI从同一任务册指出现在这一关
  → 孩子完成、换选、缩小、暂停或求助
  → 日记、作品和体验成为主线证据
  → AI校准画像、蓝图、难度和后续任务
  → 孩子确认变化，进入下一轮
```

任何功能只有在能回答以下问题之一时才应存在：

- 它是否帮助系统更准确地理解孩子？
- 它是否帮助确定或修正当前成长主线？
- 它是否帮助孩子更容易完成今天这一关？
- 它是否产生可用于下一轮调整的真实证据？

不能回答上述问题的功能，不进入主工作流。

## 4. 底部五页是同一条主线的五个阶段

底部导航统一为：`今天 / 信号 / 蓝图 / 路线 / 证据`。每页只承担一个职责，不独立生成另一套目标或任务。

| 页面 | 孩子的核心问题 | 唯一职责 | 输入到哪里 |
| --- | --- | --- | --- |
| 今天 | 我现在做什么 | 查看今日任务册并执行当前一关 | 产出完成、卡住、换选和状态证据 |
| 信号 | 最近什么让我好奇或在意 | 收集兴趣、问题、困难和灵感 | 校准画像、蓝图或主线，不直接另发任务 |
| 蓝图 | 我为什么练这些 | 确认画像、能力重点、SMART目标和OKR | 定义当前主线的方向与成功标准 |
| 路线 | 这条主线怎样推进 | 编排四周路线、本周里程碑、日程与约束 | 约束并指导今日任务册生成 |
| 证据 | 做完后AI应学到什么 | 保存体验、日记、作品和复盘 | 更新画像、蓝图、难度与下一轮计划 |

所有页面顶部持续显示同一个“当前成长主线”条：主线名称、进度、重点能力、今日完成数和五阶段位置。孩子无论在哪一页，都能知道自己正在推进同一件事。

### 明确取消的割裂逻辑

- “信号”页不再显示独立的最佳任务或候选任务池。
- “路线”页不再建立与 SMART/OKR 平行的第二套周目标。
- “今天只做这一件”不再从行动、习惯、灵感等来源独立仲裁；它只能从当天统一任务册中选择。
- 灵感、新闻和日记不是额外任务来源，而是主线规划引擎的上下文和证据。
- 能力树不是独立展示页，而是解释当前目标为什么值得练、任务在练什么。

## 5. 动态画像与未来能力树

### 三层真相

1. **孩子确认的自我描述**：最高优先级，AI 不能覆盖。
2. **可修正成长假设**：来自重复证据，显示来源、置信度、反证和更新时间。
3. **当天状态**：只影响今天的任务强度，不升级为人格或能力标签。

### 八项未来能力

| 能力 | 儿童可理解定义 | 可观察证据 |
| --- | --- | --- |
| 身心底座 | 我会按身体和情绪状态调整 | 主动恢复、合理暂停、睡眠和运动习惯 |
| 自我调节 | 我能开始、坚持、检查和收尾 | 减少提醒、完成闭环、忘记后主动补救 |
| 会学会想 | 我知道哪里不会并会换办法 | 预测、监控、错误发现、策略迁移 |
| 表达沟通 | 我能讲清意图、判断和作品 | 讲解、写作、提问、按反馈修改 |
| 数据推理 | 我用数量、模式和证据判断 | 预算、比较、记录、核验结论 |
| 创造项目 | 我把问题变成作品并迭代 | 最小版本、测试、真实成果和版本变化 |
| AI协作 | 我会使用AI并保留人的决定 | 自己先想、AI查漏、来源核验、人机分工 |
| 判断协作 | 我考虑公平、责任、来源和分工 | 角色轮换、影响检查、合作复盘 |

蓝图每轮最多突出两个能力：一个支持所有成长的底座能力，一个连接孩子兴趣与未来发展的探索能力。两个能力共同服务一条当前主线，不形成两套互相竞争的目标。

## 6. SMART、OKR 与四周路线

每条当前主线必须包含：

- 一个孩子能理解和认同的 SMART Objective。
- 最多三个可观察 Key Result。
- 四周路线，每周只说明一个主要阶段。
- 本周里程碑、家庭安排、日程和精力约束。
- 可在 2 到 15 分钟内开始的首个小实验。

SMART 约束：具体、可衡量、可做到、与孩子有意义、四周内可验证。OKR 用来判断方向是否推进，不用于排名或惩罚。若孩子不认同目标，必须先修改目标，不能用更多任务强行推动。

## 7. AI 今日任务册

### 统一生成链路

```text
已确认画像
  + 当前成长蓝图
  + 当前SMART/OKR
  + 四周路线与本周里程碑
  + 日程和今日状态
  + 近期日记与灵感
  + 历史难度、兴趣、帮助方式和动力反馈
  + 已完成任务、作品和技能关卡
  + 未来能力与学习科学知识库
  → AI生成今天唯一一份任务册
  → 从该任务册选择“现在这一关”
```

### 任务册结构

- 默认生成 24 项微任务，而不是要求孩子全部完成。
- 健康、学习、生活、责任、表达、未来六类各 4 项。
- 分为基础、成长、探索三层，孩子可按状态选择。
- 每项包含：任务、预计分钟、能力、关卡、过关标准、推荐理由和真实来源。
- AI 可以重写活动形式，但不能虚构画像、日记、新闻或证据来源。
- 当前任务卡与下方任务册使用同一个任务 ID；点击开始后必须定位并点亮同一项。

24 项的意义是提供多样性和选择空间，不是制造完成压力。首页明确告诉孩子“今天不需要全部完成”。系统优先指出一个当前关卡；孩子可以换一件真正不同的事、缩成更小版本或先照顾状态。

### 关卡难度

每项能力独立计算 1 到 5 关：

1. 接受脚手架完成。
2. 尽量独立完成。
3. 独立完成并检查。
4. 发现问题并迭代。
5. 迁移到新情境或教给别人。

升级依据包括对应技能证据、独立完成、作品迭代和“太简单”反馈；低能量、连续卡住、年龄和安全限制会临时降阶。难度不能只随总经验机械增长。

## 8. 证据回流与记忆

所有关键记录默认带有当前 `journeyId`：

- `idea.goal_id`：这条信号正在影响哪条主线。
- `action.goal_id`：行动在推进哪个目标。
- `journal.goal_id`：感悟和困惑属于哪轮成长。
- `artifact.goal_id`：作品是哪条主线的成果证据。
- `task_feedback.goal_id`：难度、兴趣、帮助和动力反馈对应哪条主线。
- `daily_mission_book.goalId`：当天任务册服务于哪条主线。

证据权重：

- 高：孩子明确更正；连续多次独立行为；真实作品的版本变化。
- 中：重复任务体验；多个场景出现同一策略；孩子确认的周复盘。
- 低：单篇日记；一次换选；家长单次观察；模型单次推断。

日记和作品由孩子逐条决定是否进入 AI 上下文。删除记录时，相关假设和蓝图证据应可撤回。健康信息只用于任务安全和强度约束，不用于诊断、能力解释或排名。

## 9. 游戏化与真实经济系统

经验、等级和宝石必须来自真实行为并持久化：

- 经验：完成任务、复盘、留下作品、验证策略后获得。
- 等级：由累计经验真实计算，不使用固定默认等级。
- 技能经验：只由对应能力证据推动。
- 宝石：用于外观、有限选择权或可见收藏，不购买能力结论。
- 随机彩蛋：低频、每日有上限、不可付费抽取、不用中断连续天数羞辱孩子。

禁止排行榜、无限连胜压力、损失厌恶文案、付费随机箱和故意拖延奖励反馈。成功不等于“做得多”，而是独立性、策略迁移、补救能力、作品迭代和自我理解提高。

## 10. 研究依据与边界

- [WEF Future of Jobs Report 2025](https://www.weforum.org/publications/the-future-of-jobs-report-2025/in-full/3-skills-outlook/)支持同时重视 AI/技术素养、创造性思维、韧性、灵活性和协作，产品不做单一编程技能树。
- [OECD Learning Compass 2030](https://www.oecd.org/en/data/tools/oecd-learning-compass-2030.html)强调学生主体性、福祉及“预期—行动—反思”，对应孩子确认画像、自主选择、行动和复盘。
- [UNESCO AI Competency Framework for Students](https://www.unesco.org/en/articles/ai-competency-framework-students)覆盖以人为本、AI伦理、技术应用和系统设计，产品要求孩子保留最终判断。
- 2024 年[元认知校准研究](https://www.nature.com/articles/s41539-024-00231-z)支持把自我监控、错误发现和策略调整纳入自我调节学习。
- 2024 年[预注册学习策略实验](https://www.nature.com/articles/s41539-024-00245-7)支持通过元认知提示改善策略选择，产品应教方法而不是只派任务。
- [How People Learn II](https://nap.nationalacademies.org/catalog/24783/how-people-learn-ii-learners-contexts-and-cultures)强调学习者、情境、文化、动机和发展阶段共同作用，系统必须读取家庭资源和真实日程。

研究边界：WEF 是雇主调查，不是儿童课程处方；单项实验不能直接证明 App 有效；所有能力权重、奖励设计和推荐策略仍需真实家庭纵向验证。

## 11. 数据与系统架构

- 账户：注册、登录、找回、监护人同意和不限数量的儿童角色。
- 主线：`growth_goals` 保存 SMART/OKR、状态和唯一当前标记 `is_primary`。
- 蓝图：`growth_blueprints` 保存证据指纹、来源、能力优先级和四周路线。
- 今日任务册：`daily_mission_books` 按角色和日期持久化，保存上下文指纹、模型来源和主线 ID。
- 证据：日记、任务反馈、作品、灵感和行动保存 `goal_id`，可沿主线聚合。
- 记忆：区分孩子确认事实、可修正假设和当天状态，并支持撤回来源。
- AI：SiliconFlow OpenAI 兼容接口，默认 `zai-org/GLM-5.2`；密钥只存在服务端环境变量。
- 回退：模型失败、超时或 Schema 错误时使用本地规则生成可解释蓝图与均衡任务册。
- 导出：按角色导出画像、主线、进度、日记授权、作品和任务证据。
- 隐私：家长简报不读取私人日记原文；图片和录音原文件默认不送入模型。

## 12. V3 迭代路线

### Sprint 1：统一主线数据模型

- 同一角色只有一个当前主线，历史方向可暂停和保留。
- 日记、作品、任务反馈、灵感和行动统一关联主线。
- 五页持续展示同一主线、进度和阶段。
- 旧数据迁移为当前主线证据或未归档证据，不丢失内容。

### Sprint 2：统一 AI 规划与今日执行

- AI 基于画像、蓝图、SMART/OKR、日程、日记和体验生成 24 项均衡任务册。
- “现在这一关”只从同一任务册中产生。
- 完成、换选、缩小和卡住反馈实时进入下一次编排。
- 显示任务推进的 Objective、KR、能力和证据来源。

### Sprint 3：闭环校准与儿童确认

- 每周生成主线变化摘要：保留、增强、减弱和未知。
- AI 根据新证据提出一个必要问题，而不是长问卷。
- 孩子确认画像和蓝图变化后才影响长期结论。
- 支持从某项结论追溯到日记、任务、作品或反馈，并可撤回。

### Sprint 4：家庭协作与规模化

- 家长空间、少催促建议、健康和安全约束。
- 可打印周卡、老人模式、日历接入和家庭资源盘点。
- 匿名化评估集覆盖年龄、安全、个性化、重复度、可执行性和证据真实性。
- 完成数据库加密、权限审计、删除/导出验收、PWA 离线和通知。

## 13. 指标与验收

北极星指标：每周由孩子独立完成至少一次“确认主线 → 执行任务 → 留下证据 → 复盘 → 接受或纠正调整”的家庭数。

- 激活：注册后完成画像确认，并建立第一条当前成长主线。
- 易用性：新用户不看教程，3 分钟内能说清“我现在在推进什么、下一步做什么”。
- 统一性：五页显示相同 `journeyId`；任务、日记、作品和反馈可追溯到同一主线。
- 任务一致性：首页当前任务 ID 必须存在于当天任务册，匹配率 100%。
- 个性化：每项成长任务至少引用一条真实上下文；AI 不得输出不存在的来源。
- 多样性：连续三天任务在活动形式、场景和产出上保持变化；高度同质推荐低于 10%。
- 成长：四周内独立开始率、闭环率、策略迁移和作品版本数上升。
- 主体性：孩子纠正画像和蓝图的入口始终可见；更正后旧结论不再参与生成。
- 安全：诊断、羞辱、成人焦虑、付费随机奖励和不适龄高风险任务进入率为 0。
- 可靠性：核心流程 E2E 全绿；模型失败时本地回退仍可完成完整闭环。

## 14. 当前不宣称

产品不宣称发现固定天赋，不预测职业，不诊断心理或健康问题，不承诺提高分数，不把相关性说成因果。V3 仍是家庭 MVP，需要至少 20 个家庭持续使用 4 到 8 周，并结合儿童访谈和行为证据，才能校准能力权重、任务密度、奖励节奏和商业假设。
